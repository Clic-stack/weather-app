# Contributing to Weather App 🌦️

First of all, thank you for considering contributing to this project! It's people like you who make the open-source community such an amazing place to learn and create.

## 🚀 How Can I Contribute?

### Reporting Bugs 🐛
If you find any UI glitches or logical errors (e.g., location not detecting or API errors not showing), please create an issue in the repository including:
* A clear and descriptive title.
* Description of the bug and steps to reproduce it.
* Mention your browser and if you provided location permissions.

### Suggesting Enhancements 💡
Do you have an idea to optimize the weather fetch or add new visual features?
* Create an **Issue** explaining the enhancement.
* If you plan to implement it, please wait for discussion before sending a Pull Request.

### Pull Requests 🏗️
1. Fork the repository.
2. Create your feature branch (`git checkout -b feature/AmazingFeature`).
3. Since this project uses **Vite** and **OpenWeatherMap**, ensure you have your `.env` file configured locally.
4. Commit your changes using **Conventional Commits** (e.g., `feat: add social login`).
5. Push to the branch (`git push origin feature/AmazingFeature`).

## ⚖️ Style Guide
* Use functional components and hooks (`useState`, `useEffect`).
* Use **Sass/SCSS** following the existing modular structure in the `styles/` folder.
* Use **Axios** for consistency with the current implementation.
* Keep the README updated if you add new features or environment variables.

# Contribuyendo a: Weather App 🌦️

En primer lugar, ¡gracias por considerar contribuir a este proyecto! Son personas como tú las que hacen de la comunidad de código abierto un lugar increíble para aprender y crear.

---

## 🚀 ¿Cómo puedo contribuir?

### Reportar Errores (Bugs) 🐛
Si encuentras fallos en la interfaz o errores lógicos (ej: la ubicación no se detecta o los errores de la API no se muestran), por favor crea un "Issue" en el repositorio incluyendo:
* Un título claro y descriptivo.
* Descripción del error y pasos para reproducirlo.
* Menciona tu navegador y si otorgaste permisos de ubicación.

### Sugerir Mejoras 💡
¿Tienes una idea para optimizar la obtención del clima o añadir nuevas funciones visuales?
* Crea un **Issue** explicando la mejora.
* Si planeas implementarla, por favor espera a que se discuta en el Issue antes de enviar un Pull Request.

### Pull Requests 🏗️
1. Haz un **Fork** del repositorio.
2. Crea tu rama de mejora (`git checkout -b feature/IncreibleMejora`).
3. Como este proyecto usa **Vite** y **OpenWeatherMap**, asegúrate de tener tu archivo `.env` configurado localmente.
4. Realiza tus **Commits** usando **Conventional Commits** (ej: `feat: add social login`).
5. Haz **Push** a tu rama (`git push origin feature/IncreibleMejora`).

## ⚖️ Guía de Estilo
* Usa componentes funcionales y hooks (`useState`, `useEffect`).
* Usa **Sass/SCSS** siguiendo la estructura modular existente en la carpeta `styles/`.
* Usa **Axios** para mantener la consistencia con la implementación actual.
* Mantén el README actualizado si añades nuevas funcionalidades o variables de entorno.
